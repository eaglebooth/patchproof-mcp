## Implementation Notes
**Approach chosen:** keep `ToolDefinition` non-generic (matches the in-source rationale in `src/tools/types.ts`); repair variance by removing the `<I,O>` from `registerTool` and typing the SDK callback via the SDK's own signature (`server.registerTool` callback's `parsed` is `unknown`-shaped per the SDK; we narrow once at the top of the handler).

**Sequenced changes (do top-to-bottom; iterate `typecheck` after each batch):**
1. **Variance + readonly fix (AC-1):**
   - `src/server/registry.ts`: drop `registerTool<I,O>` generic; type the inner callback params by destructuring the SDK's `(args, extra) => ...` signature explicitly (import `Server`/`RequestHandlerExtra` from the SDK if needed; otherwise annotate `parsed: unknown`, `_extra: unknown`).
   - `src/reporting/generator.ts`: build the result object as a mutable local `const out: { ... } = { ... }` with `html?: string` then return it typed; or split into two return branches for `html` vs non-`html` formats.
2. **Wire extra tool options through (AC-1, AC-2):**
   - Extend `AuditDependenciesInput` to include `osvMode?` and `ecosystem?` (it already does — verify they are read in the function).
   - Extend `GenerateEvidenceReportInput` with `includeHtmlPreview?`; `format?` already present.
   - Extend `AnalyzeReachabilityInput` with `includeTransitive?`.
   - Extend `PlanRemediationInput` with `riskTolerance?`.
   - Extend `RunVerificationInput` with `timeoutMs?`.
3. **Implement the 4 core tools (AC-2, AC-3):**
   - `src/scanners/files.ts`: replace empty body with a real async walk using `node:fs/promises` (`readdir({ withFileTypes: true })` + recursive descent), `ResourceGovernor` for `checkFile`/`checkBytes`/`checkDepth`, skip `DEFAULT_IGNORE_DIRS` and hidden files unless `includeHidden: true`, do not follow symlinks unless `followSymlinks: true`. Return typed counts; `findings: []` is fine (findings are produced by the dedicated tools).
   - Add `src/parsers/lockfile.ts` with `parseNpmLockfile(jsonText)` that returns a `ReadonlyArray<{name, version, isDev, isTransitive, licenses}>` from `package-lock.json` v2/v3. No external deps; pure JSON traversal.
   - `src/sbom/cyclonedx.ts`: read `<root>/package-lock.json` via `safeResolve` + `fs.readFile`, call `parseNpmLockfile`, build `components` with `purl: pkg:npm/<name>@<version>` and `licenses: [{...}]`-shaped values. Replace placeholder serial with a deterministic `urn:uuid:` derived from `SHA-256(lockfileText).slice(0,12)` formatted as a v4-shaped UUID (still purely local).
   - `src/osv/audit.ts`: add a small in-file deterministic mock `MOCK_VULNS: ReadonlyMap<string, OsvVulnerabilitySummary[]>` keyed by `name@version`. `auditDependencies` calls `parseNpmLockfile`, then for each entry looks up `MOCK_VULNS.get(key) ?? []`. `osvMode: 'live'` is accepted but the implementation reuses the mock in this run (documented in README as "live mode is reserved for a future run; mock is the only path").
   - `src/reporting/generator.ts`: keep typed empty scaffolding but now compose `findings/reachability/remediation/verification/limitations/redactions` from the empty upstream stubs (no calls into the other tools — the report just reflects the empty defaults). `html` returns a self-contained `<!doctype html><html>...<pre>{json}</pre></html>` string when `format` is `html` or `both`.

Use `fs.mkdtempSync(path.join(os.tmpdir(), 'patchproof-'))` and `fs.rmSync(..., { recursive: true })` in `afterEach`.
